# 🚀 프랜디 2026 KPI 시뮬레이터

프랜디 서비스의 2026년 매출 목표 달성을 위한 인터랙티브 시뮬레이터입니다.

## 주요 기능

### 📊 시뮬레이터 탭
- **브랜드 분포 설정**: 매장 규모별(1-50, 51-100, 101-200, 201-400, 400+) 브랜드 수와 평균 매장 수 조정
- **패키지 구성**: 5개 패키지(베이직, 프로1, 프로2, 프로3, 프리미엄)의 가격 및 브랜드 배분 설정
- **온보딩 비용**: 매장 규모별 초기 구축 비용 설정 및 포함/제외 선택
- **실시간 계산**: 입력값 변경 시 즉시 매출 시뮬레이션 결과 업데이트

### 🔀 옵션 비교 탭
3가지 전략 옵션 자동 비교:
1. **옵션1: 가격 2배 인상** - 구독료를 2배로 인상하여 수익성 개선
2. **옵션2: 가격 2.2배 인상** - 구독료를 2.2배로 인상하여 더 높은 수익 확보
3. **옵션3: 대형 브랜드 집중** - 대형 브랜드(200개 이상) 25개 확보 전략

### 📈 시각화
- 분기별 매출 추이 (Bar Chart)
- 패키지별 매출 비중 (Pie Chart)
- 매장 규모별 매출 분포 (Bar Chart)
- 분기별 브랜드 누적 현황 (Line Chart)

## 패키지 구성

### 베이직
- **포함 모듈**: AI, 메뉴정제, 대시보드
- **출시**: 1월부터 가능

### 프로1
- **포함 모듈**: 베이직 + QSC
- **출시**: 3월부터 가능 (QSC 출시)

### 프로2
- **포함 모듈**: 베이직 + 매출집계
- **출시**: 7월부터 가능 (매출집계 출시)

### 프로3
- **포함 모듈**: 베이직 + QSC + 매출집계
- **출시**: 7월부터 가능

### 프리미엄
- **포함 모듈**: All
- **출시**: 7월부터 가능

### 스탠드얼론
- **전자계약**: 독립 사용 가능

## 제품 출시 일정

- **1월**: 베이직 패키지 출시
- **3월**: QSC 모듈 출시 (프로1 가능)
- **7월**: 매출집계 모듈 출시 (프로2, 프로3, 프리미엄 가능)
- **수발주**: 2026년 출시 계획 없음

## 분기별 브랜드 유입 전략

- **Q1 (1-3월)**: 5% (제품 라인업 제한적)
- **Q2 (4-6월)**: 15% (QSC 출시 후)
- **Q3 (7-9월)**: 40% (매출집계 출시, 풀 라인업)
- **Q4 (10-12월)**: 40% (안정화 단계)

## 설치 및 실행

### 필수 요구사항
- Node.js 18 이상
- npm 또는 yarn

### 설치

```bash
cd kpi-simulator
npm install
```

### 개발 서버 실행

```bash
npm run dev
```

브라우저에서 자동으로 http://localhost:3000 이 열립니다.

### 빌드

```bash
npm run build
```

빌드된 파일은 `dist` 폴더에 생성됩니다.

### 프리뷰

```bash
npm run preview
```

## 사용 방법

### 1. 브랜드 분포 설정
- 매장 규모별 브랜드 수를 입력합니다
- 각 규모의 평균 매장 수를 조정합니다
- 온보딩 비용 포함 여부를 선택합니다

### 2. 패키지 가격 및 배분 설정
- 각 패키지의 매장 규모별 월 가격을 설정합니다
- 패키지별로 브랜드를 배분합니다
- 매장당 가격이 자동으로 계산됩니다

### 3. 결과 확인
- 총 연간 매출, 구독 매출, 온보딩 매출을 확인합니다
- 분기별 매출 추이를 검토합니다
- 패키지별, 매장 규모별 분석을 확인합니다
- 차트를 통해 시각적으로 데이터를 분석합니다

### 4. 옵션 비교
- "옵션 비교" 탭으로 이동합니다
- 3가지 전략 옵션을 한눈에 비교합니다
- 목표 매출 달성 가능성을 평가합니다

## 주요 지표

- **목표 매출**: 10억원 (연간)
- **목표 브랜드**: 45-50개
- **총 매장 수**: 약 7,000개 (브랜드 분포에 따라 변동)

## 계산 로직

### 구독 매출
```
월간 매출 = Σ(패키지 가격 × 해당 패키지의 브랜드 수)
연간 구독 매출 = 월간 매출 × 12
```

### 온보딩 매출
```
온보딩 매출 = Σ(매장 규모별 온보딩 비용 × 신규 브랜드 수)
```

### 분기별 매출
- 제품 출시 일정에 따라 패키지 가용성 반영
- 분기별 브랜드 유입률 적용
- 누적 브랜드 수 기반 구독 매출 계산
- 신규 브랜드 기반 온보딩 매출 계산
